<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: DAW sidechain validation</name>
  <what-built>
    Canary VST3 plugin with sidechain bus configuration and audio passthrough. The plugin should:
    - Load in DAW plugin lists
    - Show sidechain input routing options
    - Pass main input audio through to output unmodified
    - Not crash when no sidechain is connected
  </what-built>
  <how-to-verify>
    Test in at least ONE DAW (ideally Ableton first, then others):

    **Ableton Live:**
    1. Open Ableton Live
    2. Go to Preferences > Plug-Ins and rescan VST3 plugins if needed
    3. Create an audio track with some audio content (voice or any audio)
    4. Insert Canary (VST3) on the track
    5. Verify the plugin loads and shows a black window with "Canary" text
    6. Play audio -- verify it passes through unchanged (passthrough)
    7. Click the plugin's sidechain panel toggle (wrench icon or sidechain section)
    8. Verify a dropdown appears listing other tracks as sidechain sources
    9. If available, route a synth/pad track as sidechain source (for Phase 1, just confirm the dropdown exists)

    **Reaper (alternative):**
    1. Insert Canary on a track
    2. Open the plugin's I/O routing (click the pin connector button)
    3. Verify 4 input pins are shown (2 main + 2 sidechain)
    4. Play audio -- verify passthrough works

    **FL Studio (alternative):**
    1. Load Canary on a mixer insert
    2. Verify the mixer routing matrix shows sidechain routing option
    3. Play audio -- verify passthrough

    **Logic Pro (optional -- VST3 sidechain support may be limited):**
    1. Insert Canary VST3 on a track
    2. Check for sidechain dropdown in plugin header
    3. Document result (even if sidechain is absent -- this is expected for VST3 in Logic)

    **Critical checks:**
    - [ ] Plugin loads without error in at least one DAW
    - [ ] Sidechain routing option is visible in at least one DAW
    - [ ] Audio passes through when playing (no silence, no distortion)
    - [ ] Plugin does not crash when no sidechain is connected
  </how-to-verify>
